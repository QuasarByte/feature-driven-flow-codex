# Release Readiness Gate Template

Use when `release-readiness-policy` is active.

## Change Summary

1. What is changing?
2. Why is it changing?
3. What is the expected impact?

## Rollout

1. Rollout strategy (staged? feature flag? percent rollout?).
2. Preconditions / migrations.
3. Monitoring signals during rollout.

## Rollback

1. Rollback plan.
2. Data rollback considerations.
3. Backward compatibility status.

## Observability

1. Key logs/metrics/traces to watch.
2. Alert thresholds and ownership.
3. Known blind spots.

## Go/No-Go Recommendation

1. Recommendation: `go|no-go|go-with-constraints`
2. Rationale.
3. Constraints / follow-ups.
